# Step 1: Read the farm's memory first

**Progress: Step 1 of 8** — Next: Check the sanctum still matches the save

**Read the farm's plan first: `plans/PLAN.md`.** This is the single living plan — the Current
focus (where the farm stands and what's next) plus the Standing directives it is built from. It
is the one file that tells you what this farm is actually trying to do, so it is read before
anything else, alongside `identity/creed.md` which sets the voice. Do not skip it, and do not
fold it into a quick scan of the other files — a briefing that has not read the plan is guessing
at the farm's intent, and the whole reason the plan is written down is so it does not have to.

Then the rest of the farm's memory: `sanctum/config.json`, `identity/creed.md`,
`history/closeout-latest.md`, and `identity/decision-making.md`, plus `references/decision-making.md`
(the general policy — short, read it in full). Note anything under "What didn't work" in the
closeout: **don't re-recommend what was tried and rejected last session.** That file exists
precisely so the same mistake doesn't get made twice.
